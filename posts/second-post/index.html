Why is math so abstract

Motivation

Have you ever wondered why is math so abstract? In this article I want to dive deeper into this question and give you some examples to demonstrate what actually is abstraction in math and why it is important. But first… let’s make clear what we mean by word abstract.

What is abstract??

Abstract thing is something that only exist as a thought or an idea. It doesn’t exist in the real world.

We can take a line as an example.
In mathematics line is an infinitely long perfectly straight object. If we want to be a bit fancier, instead of straight we can say one-dimensional.

Something like that definitely doesn’t exist. It is an idealisation. It isn’t even tangible so how could it be. It is infinite also which is a bit problematic too.
Surely we can imagine some connection between two points and imagine that somehow it continues forever, but that’s exactly the abstractness of it. It’s just an idea. In reality there isn’t anything connecting those two points.

Same goes for vectors in physics. In reality there isn’t any arrow pulling you down towards Earth, it’s just a way to visualize this force.

Of course these are just some basic examples. As you’re getting to higher level maths, everything becomes more and more abstract and there is always some point when the abstraction stops making sense.

Therefore… If you ask someone about usefulness of math, often times common idea that comes to anyone’s mind is that math is useless cause of all the abstractness. Like if math only cares about some abstract unreal ideas. How is it relatable and useful to a real world problems? And sure it sounds logical. But… only if you don’t think it deeper.

Actually, it is quite the exact opposite. If we would want math to be more real world relatable subject, it wouldn’t be as useful as it is. Or you can say that it naturally becomes more and more abstract as people want it to be more and more useful, even to the stage it stops being any related to real world

So usefulness of math lies in its abstractness. But how? Everything I’ve written so far is still very theoretical. So let’s explore some more concrete examples.

First and second level of mathematical abstraction

In elementary school there are two really important abstraction levels you need to achieve. But a lot of people doesn’t realize they are abstractions or even they need to think about them that way.

First level

First level comes right in the first grade or probably even earlier.
Every children is taught that if you have four apples and you gain another two apples. You just have six apples.
But same goes for example with age. When you’re four years old, you know that two years later you will be six years old.

I’m sure we would agree that “apples” and “years” are really different things/objects, no need to mention the differences, but when it comes to counting them, it’s somehow the same concept. This is important. Up until now, it only made sense to talk about quantity of some specific objects. But we can separate the quantity from those objects and talk about it itself.
When you realize this, numbers doesn’t describe anything real anymore. They are an abstract ideas describing quantity. So for example we can think of number 3 the way that somehow, it describes any possible triplet you could imagine. This is called first level of mathematical abstraction.

Second level is a bit more interesting and much more important. It comes in eighth grade in our country. Numbers have been well rooted in your head as an idea of quantity and you’ve learned how to manipulate them pretty much in school. That manipulation with numbers is called arithmetics.

Time has come to get to another level of abstraction… variables.

Second level

Variables are abstraction of numbers same as numbers are abstraction of quantity. When we have some variable \(a\) or \(x\) for example. We think of it as it represents any number. We don’t care about concrete values anymore.

At first it may not be clear how this is any useful. So hold with me for a while.
Before we start, what’s important to realize is that somehow all numbers are contained in that one variable, any possible number you can imagine. So if numbers are abstraction of quantity of some objects. variables are more like abstraction of quantity itself.

Theory of algebra was developed on that abstraction and I feel like a lot of people doesn’t realize how algebra is actually really useful, because they never really understood variables and what they represent.

So the key to understand how useful it is… Imagine we have some real world problem that can be abstracted/represented, described and solved using numbers. Then of course it can be represented using variables since variables are just abstraction of numbers.

The issue with numbers is that even if you have basically the same type of problem, you need to solve it again and again everytime cause the quantities/values have probably changed. Speed of car for example. Or maybe you can remember the way to solve it, but for more complicated hard to memorize problems you would probably just reinvent the wheel everytime again. But there is a clever way to avoid this.

As we said earlier variables somehow represents any number you could have. So if we were able to solve the problem represented by variables, then the same solution applies to any values we start with. This way we only need to solve one type of problem once and just plug in the values in the end. And that’s exactly what algebra is all about, solving problems using variables and then simplifying to the most simple solution.

That abstract solution that applies to any problem of the same type is called general solution. And generality is exactly what we’re searching for in mathematics. For better understanding, let’s demonstrate how to use variables with one example.

Compound interest

Let’s assume you have \( 1 000 \$ \) saved on your savings account with annual interest rate \( 5\% \). Don’t think about any taxes to keep it simple. The question is how much money you will have after 3 years.

After first year you will have your original \( 1 000 \$ \) increased by \( 5\% \) of your money. So that’s $$ 1000+ \frac{5}{100}\cdot1000 = 1050\$ $$

After second year you already have \( 1 050 \$ \). So your final amount will be $$ 1050+ \frac{5}{100}\cdot1050 = 1102.5\$ $$

And finally after third year

$$ 1102.5+ \frac{5}{100}\cdot1102.5 = 1157.625\$ $$ You probably see this is really long not practical approach to solve this problem. So let’s try it differently. I’ll rephrase the same problem so it is represented using variables. Feel free to match them to see how similar they are, realising that second version is just a bit more abstract.

Let’s assume you have \( M_0\) saved on your savings account with annual interest rate \( I \). The question is how much money you will have after \(n\) years.

Index of \(M\) states how many years have passed. So if we want to know how much money we have after \(n\) years, we’re basically searching for \(M_n\). We will now do the same steps as before.
After first year your money will be. $$ M_0+ \frac{I}{100}\cdot M_0 = M_1 $$ We can rewrite this using algebra as $$ M_0+ \frac{I}{100}\cdot M_0 = M_0 \left(1 + \frac{I}{100}\right) = M_1 $$ After second year it will be $$ M_1+ \frac{I}{100}\cdot M_1 = M_1 \left(1 + \frac{I}{100}\right) = M_2 $$ and so on…

Notice that what we’re actually doing is just each year multiplying by factor \(\left(1 + \frac{I}{100}\right)\). This means that $$ M_n = M_0 \cdot \underbrace{\left(1 + \frac{I}{100}\right)\cdot \left(1 + \frac{I}{100}\right)\cdot\cdot\cdot \left(1 + \frac{I}{100}\right)}_{n \text{ times}} = M_0 \cdot \left(1 + \frac{I}{100}\right)^n$$ This is general solution to our problem. Let’s try if it works by plugging in the values from the original problem. $$M_3 = 1000 \cdot \left(1+\frac{5}{100}\right)^3 = 1000 \cdot (1 + 0.05)^3 = 1000\cdot1.05^3= 1157.625\$ $$ Solutions matches, everything works just fine.

So this was an example of abstraction. You probably have some idea already, but let’s talk about how abstraction and generality are connected.

Abstraction as a way to generality

Actually, the same concepts of abstraction are used commonly in higher mathematics. I guess we’re ready now. Finally it is time to define what abstraction is.

So what we mean by abstraction in mathematics is looking at some different objects and try to figure out something that they have in common. In other words to find some abstract connection between those objects. Then use that abstract connection and build a theory about it.

Examples

  • algebra for variables
  • arithmetics for numbers

And finally use that theory to find some facts about these objects which result only from those properties they have in common.
This is why abstraction is so important. Not only that we can find general facts about some system of objects which are only related with our abstract connection. But also in a certain way it simplifies the problems cause we ignore all the concrete facts about these objects that are not important to us. Pretty cool, don’t you think?

There are a lot of important abstractions and theories about them.
For example graph theory which is a general way to describe relations or group theory which is a general way to describe symmetry.

Or really important third level of mathematical abstraction, theory of vector spaces, which says that basically any object can be represented as some n-dimensional vector. Dimension depends on what type of object we want to represent.
That’s a little hint why mathematicians care about higher dimensions. It’s not really about vectors themselves. It’s about what they represent.

For example… did you know that modern translators are representing words as vectors using something called Word2vec algorithm made by czech computer-scientist Tomas Mikolov? Even translating can be abstracted using vector spaces and there is many more ways to apply them.

Conclusion

So I hope I have answered the question well enough. Despite the fact that abstraction is a basis of mathematics and there are tons of examples why it is true. They are pretty hard to explain since they are abstract of course. But I hope I somehow captured the main idea.
